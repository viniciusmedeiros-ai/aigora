A OpenAI anunciou uma nova iniciativa na segunda-feira projetada para ajudar a comunidade de código aberto a melhorar seu jogo de segurança cibernética e evitar bugs.

"Patch the Planet" (que é uma alusão não tão sutil a " Hack the Planet", o icônico slogan do filme Hackers, de 1995) fará com que a OpenAI se junte à empresa de segurança Trail of Bits  para ajudar os mantenedores de código aberto a proteger seus projetos.

A OpenAI disse que a equipe de segurança da Trail of Bits trabalhará diretamente com os mantenedores de código aberto para revisar possíveis problemas de código. As ferramentas de segurança da OpenAI — como o Codex Security — serão usadas para auxiliar no processo.

"Muitos mantenedores já estão sendo solicitados a classificar mais relatórios, mais rapidamente, com o mesmo tempo e recursos limitados", disse a OpenAI na segunda-feira. “O Patch the Planet é construído para reduzir essa carga, não para aumentá-la: os engenheiros de segurança analisam as descobertas antes de chegarem aos mantenedores, trabalham com projetos para desenvolver patches e testes e criam fluxos de trabalho reutilizáveis que ajudam as equipes a continuar melhorando segurança após as primeiras correções de terra.”

Em outras palavras, os engenheiros da Trail of Bits funcionarão mais ou menos como EMTs de código — para ajudar os mantenedores de projetos de código aberto a identificar e triar possíveis problemas, todos suportados pelo software da OpenAI. Parece um projeto ambicioso, e não está claro como ele funcionará a longo prazo, ou como planeja expandir (se for o caso).

Os projetos de código aberto são a base digital sobre a qual a indústria de software comercial se apoia, mas, infelizmente, devido à estrutura descentralizada e mal monitorada desse ecossistema, grande parte do software é insegura. Bugs em projetos de código aberto podem se transformar em grandes problemas para bases de código comerciais. O desastre log4j de vários anos atrás — quando uma vulnerabilidade ruim foi descoberta em um utilitário de código aberto amplamente utilizado — é um bom exemplo.

Grande parte da preocupação em torno de ferramentas como o Mythos (a ferramenta de segurança altamente divulgada da Anthropic) parece derivar do fato de que a IA agora pode identificar automaticamente os bugs existentes nas bases de código e começar a criar exploits para eles. Embora a automação do cibercrime não seja nova, essas ferramentas, sem dúvida, têm o potencial de torná-la significativamente mais conveniente para os maus atores.

A OpenAI está virando essa fórmula de cabeça para baixo ao usar a IA para ajudar a comunidade de código aberto a se proteger melhor. É difícil não lê-lo como um golpe competitivo no Anthropic, ao mesmo tempo em que reconhece que é algo que a comunidade de código aberto precisa desesperadamente.
