Os arreios de agentes estão se tornando a maneira dominante de construir agentes, e eles não vão a lugar nenhum. Esses arreios estão intimamente ligados à memória do agente. Se você usou um arnês fechado - especialmente se estiver por trás de uma API proprietária - você está optando por ceder o controle da memória do seu agente a terceiros. A memória é incrivelmente importante para criar experiências agênticas boas e pegajosas. Isso cria um bloqueio incrível. A memória - e, portanto, os arreios - devem estar abertos, para que você possua sua própria memória

A "melhor" maneira de construir sistemas agênticos mudou drasticamente nos últimos três anos. Quando o ChatGPT foi lançado, tudo o que você podia fazer eram correntes de PANO simples ( LangChain ). Então os modelos ficaram um pouco melhores, e poderiam criar fluxos mais complexos ( LangGraph ). Então eles melhoraram muito, e isso deu origem a um novo tipo de andaime - arneses de agente.

Exemplos de chicotes de agente incluem Claude Code , Deep Agents , Pi (poderes OpenClaw), OpenCode , Codex , Letta Code e muitos mais.

Às vezes, há um sentimento de que os modelos absorverão cada vez mais o andaime. Isso não é verdade. O que aconteceu (e continuará a acontecer) é que muitos dos andaimes necessários em 2023 não são mais necessários. Mas isso foi substituído por outros tipos de andaimes. Um agente, por definição, é um LLM interagindo com ferramentas e outras fontes de dados. Sempre haverá um sistema em torno do LLM para facilitar esse tipo de interação. Precisa de provas? Quando o código-fonte do Claude Code vazou, havia 512 mil linhas de código . Esse código é o arnês. Mesmo os fabricantes do melhor modelo do mundo estão investindo pesadamente em arreios.
