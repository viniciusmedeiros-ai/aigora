sqlite AGENTS.md ( via ) O SQLite ganhou um arquivo AGENTS.md há cinco dias - mas não é destinado ao seu próprio desenvolvimento, é presumivelmente destinado a pessoas que estão apontando agentes para a base de código SQLite. Ele inclui:

A SQLite não aceita solicitações pull sem acordo prévio e/ou documentação legal que a acompanha que coloca a solicitação pull no domínio público. No entanto, os desenvolvedores humanos do SQLite revisarão uma solicitação pull concisa e bem escrita como uma prova de conceito antes de reimplementar as próprias alterações.

O SQLite não aceita código agentic. No entanto, o projeto aceitará relatórios de bugs autênticos que incluam um caso de teste reproduzível. Patches ou pull requests demonstrando uma possível correção, para fins de documentação, são bem-vindos.

O commit mais recente para esse arquivo removeu a palavra "(currently)" de "SQLite does not accept agentic code, with the commit message "Strengthen the statement about not accept agentic code".

Enquanto isso, o fórum do SQLite estava sendo inundado com tantos relatórios de bugs gerados por IA - de qualidade variável - que agora eles os dividiram em um novo Fórum de Bugs do SQLite. D. Richard Hipp está resolvendo problemas com uma enxurrada de commits na base de código.
